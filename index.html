The Challenges of Modern Front-end Development

The technologies of the Web

  • The languages of the web are HTML, CSS and JavaScript
  • Ensuring browser support is necessary because they are standards sith multiple independent implementations
  • Modern browsers are mostly standards complian

The early Web

  • An information system where documents may be interlinked
  • Developed between 1989 and 1990 by English scientist Tim Berners-Lee at CERN
  • A markup language originally intended to enrich plain text documents with hypertext
  • The first website is still hosted on CERN's website
  • It can also be browsed in a simulator of the line-mode browser

Early browsers

  • No standard for layout and styling until 1996
  • HTML served for formatting the document
  • Advanced styling would be done through browser hacks, table layouts and embeded flash content
  • Browser manufacturers extended HTML with unstandartized presentational tags

CSS

  • Separation of concerns between HTML and CSS
  • Proposed by HÃ¥kon Wium Lie in 1994
  • CSS specification achieved its current form in 1996
  • First browser to feature CSS Level 1 support was Internet Explorer 5 for Macintosh, shipped in 2002

CSS adoption

  • Internet Explorer 6 was released in 2001 with good CSS Level 1 support and 80% of the market share
  • Two major website redesigns - of Wired and ESPN in 2002/2003 took a chance on the new standard
  • CSS Zen Garden, published in 2003, aimed to encourage the use of CSS alone for styling and layout purposes

The changing Web

  • The Web is not an information system but a generic programing platform now
  • Websites are accessed through a huge variety of devices and users (and not only users)
  • HTML5 introduced semantic tags which communicate intent rather than format and aid accessibility

Web applications

  • The content on the Web is also transforming
  • Server-side rendering was replaced by AJAX which opened the door for Single Page Applications
  • Users expect sites to be interactive and responsive, web applications have to support user input
  • The advent of JavaScript frameworks

Business logic and presentational logic

  • As apps become more complex and interactive the UI elements also become more complex
  • Shift towards more app-oriented Web technologies
  • The concept of reusable UI components has even given rise to the Web Components Web standard
  • Complex internal DOM that is not exposed
  • CSS now supports scoped variables

Product needs

  • In a saturated market every product needs a successful brand
  • Consistent design and user experience across platforms
  • UX and UI designers have to work closely with Web developers and share a common vocabulary to achieve a successful user experience
  • Detail and bigger picture: component and flow

Front-end libraries

  • Some early attempts to outsource the UI development have been CSS helper themes
  • General purpose libraries like Bootstrap and Material Design
  • Currently such component libraries are based on Design Systems
  • Brands and products also develop their own

Component library advantages

  • Reusability
  • Modification of the design and behaviour is (relatively) easy
  • Out of the box building blocks that do not interfere with business logic
  • Responsive and accessible, can handle the needs of a contemporary web app
  • Tested and standards compliant
  • Easier designer-developer hand off

Contemporary Front-end developers

  • Have to know the technologies in depth HTML, CSS and JavaScript
  • Write clean and performant code
  • The three core technologies of the web, HTML, CSS and JavaScript are now very powerful and keep having new features
  • The future of the web is bright and accessible
